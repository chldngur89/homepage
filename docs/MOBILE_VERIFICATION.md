# 모바일 반응형 검증 체크리스트

## 코드 기준 확인 완료

- **viewport**: `index.html`에 `width=device-width, initial-scale=1.0` 설정됨
- **네비게이션**: 768px 미만에서 햄버거 메뉴, 이상에서 가로 메뉴
- **터치 타겟**: CTA·메뉴 버튼 `min-h-[44px]` 이상 적용
- **레이아웃**: 전 페이지 `max-w-*`, `px-6`, `grid-cols-1 md:grid-cols-*` 등 반응형 클래스 사용
- **가로 스크롤 방지**: Layout 루트에 `overflow-x-hidden` 적용

## 직접 확인 방법

### 1. 로컬에서 모바일 뷰로 보기

```bash
npm run dev
```

브라우저에서 개발 서버 접속 후:

- **Chrome**: F12 → 상단 **Device Toolbar** 아이콘(휴대폰 모양) 클릭 → 기기 선택 (iPhone 14 Pro, Galaxy S20 등) → 새로고침
- **Safari**: 개발자 메뉴에서 "반응형 디자인 모드" 또는 실제 기기 연결

### 2. 빌드 후 프리뷰

```bash
npm run build && npm run preview
```

나온 URL을 **실제 스마트폰**에서 접속 (같은 Wi‑Fi에서 PC IP로 접속, 예: `http://192.168.x.x:4173`).

### 3. 확인할 항목

- [ ] 상단 로고·햄버거 메뉴가 잘 보이고, 메뉴 열기/닫기 동작
- [ ] 히어로 제목·설명이 줄바꿈되고 가로 스크롤 없음
- [ ] 4단계 플로우 카드가 세로로 쌓임 (1열)
- [ ] 앱 페이지 카드가 1열로 표시, 긴 앱 이름 줄바꿈
- [ ] 푸터 링크·텍스트가 읽기 좋게 정렬
- [ ] CTA 버튼 터치 시 잘 클릭됨 (작은 영역 없음)

## 빌드 검증

`npm run build` 성공 시 프로덕션 번들이 정상 생성된 것이며, Vercel 등에 배포 후 실제 모바일에서 한 번 더 확인하는 것을 권장합니다.
