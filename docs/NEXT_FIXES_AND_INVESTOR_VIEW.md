# 다음에 고칠 것 & 투자자·심사관 관점

## 1. 로컬에서 잘 돌아가게 (현재 상태)

- **빌드**: `npm run build` ✅ 성공 (로컬에서 이미 잘 돌아감)
- **개발**: `npm run dev` → http://localhost:5173 에서 수정·테스트 후 Vercel에 push 하면 됨

**추가로 손보면 좋은 것 (선택)**
- 일부 페이지에서 `<a href="/demo">` 처럼 **일반 링크**로 되어 있으면, SPA라서 클릭 시 전체 새로고침이 날 수 있음. 내부 이동은 `<Link to="/demo">` 로 바꾸면 더 부드러움 (Solution, About, Pricing 등).
- `index.html` 제목이 아직 "아이디어 회의 진행" → **"ZeroSeller"** 등 서비스명으로 바꾸면 탭·검색·공유 시 첫인상이 좋아짐.

## 2. 우리가 뭘 더 고치거나 수정해야 할까? (우선순위)

### 🔴 우선 (신뢰·첫인상)
| 순서 | 항목 | 이유 |
|------|------|------|
| 1 | **페이지 제목·메타** | `index.html`의 `<title>`, `<meta name="description">`를 "ZeroSeller – 대표님은 결정만 하세요" 식으로 수정. 공유·검색·첫인상에 직결. |
| 2 | **사회적 증거** | 고객 후기(구체 수치), 고객사/파트너 로고가 없으면 "실제로 쓰는 곳이 있나?" 싶어짐. 베타 테스터 1~2명이라도 인용 가능하면 좋음. |
| 3 | **푸터 정책 링크** | 개인정보처리방침·이용약관이 `#` 이면 "준비 안 된 회사"로 보일 수 있음. 최소한 페이지 하나씩 만들거나, PDF/노션 링크라도 넣기. |

### 🟡 그다음 (전환·운영)
| 순서 | 항목 | 이유 |
|------|------|------|
| 4 | **문의 폼 실제 연동** | 지금은 제출해도 서버로 안 감. Formspree 등으로 이메일/Slack 연동하면 IR·문의가 실제로 들어옴. |
| 5 | **데모 페이지** | "무료 체험" 클릭 시 **무엇을 보게 되는지** 1~2문장으로 명시 (예: "대시보드 체험 링크" / "영업 연락 요청")하면 심사관·투자자가 다음 행동을 예측하기 쉬움. |
| 6 | **연락처** | 푸터·문의 페이지의 전화번호·주소·이메일이 placeholder면 실제 연락처로 교체. |

### 🟢 여유 있을 때
- 모바일에서 버튼·폼 터치 영역, 스크롤 후 CTA 노출(sticky) 확인
- GA4 등으로 유입·전환 측정 (개인정보처리방침에 반영)
- 콘텐츠 JSON 분리, 챗봇 등

## 3. 투자자·심사관이 뭐라고 할 것 같아?

아래는 **예상 코멘트**입니다. (좋은 점 + 지적받기 쉬운 점)

### 👍 잘 보일 부분
- **컨셉이 뚜렷함**: "Zero-Click Intelligence", "대표님은 결정만" 같은 한 줄 메시지가 있어서 10초 안에 뭘 하는 서비스인지 전달 가능.
- **기술·차별화 언어**: Self-Reflecting Strategy Loop, AI 마켓 가디언, PCT 특허 등 기술 키워드가 있어서 기술 심사·IR에서 이야기할 거리가 있음.
- **구조**: 홈·솔루션·기술·요금·데모·회사소개·문의까지 페이지 구성이 정돈되어 있음.
- **디자인 톤**: 다크·미니멀하게 잘 맞춰져 있어서 "제품 수준"으로 보이기 좋음.

### ⚠️ 이렇게 지적받기 쉬운 부분

| 지적 예시 | 의미 | 대응 방향 |
|-----------|------|------------|
| **"실제 사용 사례가 없어 보인다"** | 후기·로고·수치가 없어서 검증이 안 된 것처럼 보임. | 베타 사용자 인용, 파트너/지원사업 로고, "예비창업패키지 진행 중" 등이라도 명시. |
| **"문의해도 연락이 갈지 모르겠다"** | 폼이 동작 안 하거나, 연락처가 가짜처럼 보임. | 폼 연동 + 실제 이메일/전화 기재. |
| **"회사/팀이 안 보인다"** | About에 팀·대표 소개가 약하면 신뢰가 떨어짐. | 대표·핵심 팀 1~2명, 경력 한 줄이라도. |
| **"법적 보호가 준비됐는지 모르겠다"** | 개인정보처리방침·이용약관이 없거나 # 이면 리스크로 봄. | 최소한 1페이지씩 또는 외부 링크라도 제공. |
| **"제목이 '아이디어 회의 진행'인데 서비스가 ZeroSeller인가?"** | 탭·공유 시 제목이 Figma 프로젝트명으로 나오면 혼란. | `<title>`을 서비스명·한 줄 설명으로 변경. |
| **"데모/체험이 뭔지 모르겠다"** | "무료 체험"만 있고, 클릭 후 무엇을 보는지 없음. | 데모 페이지에 "체험 링크" 또는 "영업 연락 요청" 등 1~2문장 설명. |

### 💬 심사관·투자자 입장 요약
- **"메시지와 기술 스토리는 좋은데, 검증(트랙션·고객)과 법·연락이 보강되면 더 설득력이 있다"**  
- **"첫 화면과 2~3분 둘러보는 수준에서는 괜찮은데, 문의·정책·팀 정보까지 보면 아직 준비 중인 느낌이 든다"**

## 4. 정리: 지금 단계에서 할 일 추천

1. **즉시**: `index.html` 제목 + 메타 설명을 ZeroSeller 기준으로 수정. ✅
2. **1주 내**: 푸터 개인정보처리방침·이용약관 링크를 실제 페이지 또는 문서 링크로 연결. ✅ (페이지 추가 완료)
3. **가능하면**: 고객/파트너 인용 1~2개 또는 "예비창업패키지·청년창업사관학교 지원 예정" 등 실체 있는 문구 추가. ✅ (홈에 후기·파트너 섹션 추가)
4. **그다음**: 문의 폼 연동(Formspree 등) + 연락처·데모 설명 보강. ✅ (Formspree 연동 구조 + 데모 설명 + 연락처 안내 문구)
5. **여유**: 스티키 CTA(모바일), 터치 영역(48px), GA4 placeholder, 콘텐츠 JSON 예시. ✅ (모두 반영)

이렇게 하면 **로컬에서 잘 돌아가는 건 유지한 채**, 투자자·심사관이 볼 때 "준비된 팀"으로 보이기 훨씬 좋아집니다.
