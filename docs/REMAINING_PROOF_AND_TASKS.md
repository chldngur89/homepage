# 남은 것: 증명(검증) vs 추가로 해야 하는 것

웹/IR 기준으로 **이미 한 것**을 제외하고, **증명**과 **추가 작업**만 정리했습니다.

---

## 1. 증명하거나 보강해야 하는 것 (Investor / 심사관이 “검증”으로 보는 것)

| 항목 | 현재 상태 | 증명/보강 방향 |
|------|-----------|----------------|
| **실제 사용·트랙션** | 후기·파트너 문구는 **플레이스홀더** (예: "강원 소상공인 A", "예비창업패키지") | 실제 베타 사용자 1~2명 동의 받고 **실명·직함(또는 익명 but 역할)** + **구체 수치**로 교체. 예: "OO 대표, 전환율 40% 상승" |
| **팀·대표** | About에 팀/대표 소개가 **없거나 약함** | 대표·공동대표 1~2명 **이름 + 한 줄 경력** (또는 "팀 소개" 블록). 사진은 선택. |
| **연락처 신뢰** | 이메일·전화·주소가 **샘플 값** (contact@zeroseller.ai, 010-1234-5678 등) | 실제 사용할 **이메일·전화**로 교체. 주소는 "문의 시 안내"로 두어도 됨. |
| **문의가 실제로 전달됨** | Formspree **연동 구조만** 있음 (ID 미설정 시 제출해도 이메일로 안 감) | Formspree(또는 대안)에 **실제 Form ID 등록** 후 `.env` 설정 → 문의가 실제 메일로 전달되는지 테스트. |
| **제품이 존재함** | 데모는 **시뮬레이션**만 있고, 실제 대시보드 UI 없음 | 실제 **대시보드/앱 스크린샷** 1~2장 확보 후, 홈/솔루션/데모에 "실제 화면" 캡처로 교체. 없으면 "출시 예정" 등으로 명시. |
| **지원사업·파트너** | "예비창업패키지", "청년창업사관학교", "TIPS 예정" **텍스트만** 있음 | 실제 **선정/지원 내역** 있으면 "예: 2025 예비창업패키지 선정" 등으로 구체화. 없으면 "지원 예정" 유지. |
| **특허·IP** | "PCT 출원 준비 중" 문구만 있음 | 실제 **출원 번호/국가** 있으면 표기. 없으면 "준비 중" 유지. |

**정리**: 웹만 보는 심사관·투자자는 위 항목이 **실제 데이터**로 채워져 있을 때 "검증된 팀/서비스"로 인식합니다. 우선순위는 **연락처·문의 전달** → **팀/대표** → **실제 후기·수치** → **제품 스크린샷** 순으로 추천합니다.

## 2. 추가로 해야 하는 것 (운영·기술·콘텐츠)

### 2-1. 운영·설정 (한 번만 하면 됨)

| 할 일 | 설명 |
|-------|------|
| **Formspree Form ID 설정** | formspree.io에서 폼 생성 → Form ID를 `.env`의 `VITE_FORMSPREE_FORM_ID`에 넣고 배포 환경(Vercel 등)에도 동일 설정. |
| **도메인 연결** | 원하면 zeroseller.ai 등 **커스텀 도메인**을 Vercel 프로젝트에 연결. (현재 homepage-roan-kappa.vercel.app 만 있어도 동작함) |
| **실제 연락처 반영** | Contact 페이지·푸터(필요 시)의 이메일·전화번호를 **실제 사용할 값**으로 수정. |

### 2-2. 콘텐츠·자산

| 할 일 | 설명 |
|-------|------|
| **후기·로고를 실제 데이터로** | 홈 "검증과 신뢰" 섹션의 후기 2개를 **실제 동의 받은 인용**으로 교체. 파트너/지원사업은 **실제 명칭·선정 여부**로 수정. |
| **대시보드/앱 스크린샷** | 실제 UI가 나오면 **인사이트 카드·승인 대기 리스트**가 보이는 화면을 캡처해 홈/솔루션/데모에 사용. |
| **About 팀/대표** | 대표·핵심 멤버 1~2명 **이름 + 역할 + 짧은 경력** (필요 시 사진) 추가. |
| **정책 문구 검토** | 개인정보처리방침·이용약관은 **법률 검토**가 있으면 좋고, 서비스 확장 시 수집 항목·목적 등만 최신화해도 됨. |

### 2-3. 측정·개선 (선택)

| 할 일 | 설명 |
|-------|------|
| **GA4 연동** | `index.html`의 GA4 주석 해제 후 **측정 ID** 입력. 유입·전환 확인용. (개인정보처리방침에 이미 분석 도구 안내 있음) |
| **모바일 실제 기기 점검** | 실제 폰에서 스크롤·폼·스티키 CTA 동작 확인 후 필요하면 수정. |
| **로딩 속도** | 이미지 WebP·번들 크기 등 필요 시 점검. (현재 빌드 용량은 무난한 수준) |

## 3. 체크리스트 (현재 기준)

**증명·검증**
- [ ] 문의 폼이 실제 이메일로 전달됨 (Formspree ID 설정 + 테스트)
- [ ] 연락처(이메일·전화)가 실제 사용할 값으로 표기됨
- [ ] About에 대표/팀 1~2명 소개 있음
- [ ] 후기 1~2개가 실명/역할 + 구체 수치로 되어 있음 (또는 “출시 후 반영” 등 명시)
- [ ] 실제 대시보드 스크린샷이 있거나, “출시 예정” 등이 명확함
- [ ] 지원사업/파트너는 실제 명칭·선정 여부로 정리됨

**추가 작업**
- [ ] Formspree Form ID 설정 및 배포 환경 반영
- [ ] 도메인 연결 (원할 경우)
- [ ] GA4 등 분석 도구 사용 시 ID 입력 및 개인정보처리방침 유지

---

## 4. 요약

- **증명**: “실제로 동작하고, 실제 사람/팀이 있고, 실제로 쓰는 사람이 있다”를 보여주는 것.  
  → **연락처·문의 전달** → **팀 소개** → **실제 후기·수치** → **제품 스크린샷** 순으로 채우면 됩니다.
- **추가로 할 일**: 위 증명을 채우기 위한 **설정(Formspree, 연락처)** 과 **콘텐츠(후기, 팀, 스크린샷)** 반영.  
  도메인·GA4·모바일 점검은 여유 있을 때 진행하면 됩니다.

이 문서를 기준으로 하나씩 체크해 나가면, “남은 것”을 명확히 관리할 수 있습니다.
