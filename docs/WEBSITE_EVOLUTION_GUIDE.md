# ZeroSeller 웹사이트 진화 가이드
## 고객이 원하는 것 & 우리가 할 일

여러 검색·자료를 바탕으로 **고객이 실제로 원하는 것**과 **우리 사이트에 적용할 수 있는 액션**을 정리했습니다.

---

## 1. 고객이 원하는 것 (검색 기반 요약)

### 1-1. 첫인상 & 메시지 (3~10초 안에 결정)
- **"이게 나한테 필요한지"** 10초 안에 판단한다.
- **기능 나열보다 혜택**이 중요하다.  
  예: "클라우드 저장" ❌ → "어디서든 파일 접근" ✅
- **명확한 CTA(행동 유도)** 하나가 있으면 전환율이 최대 4배까지 올라간다.
- 문장은 **초등 5~7학년 수준**이면 이해도·전환율이 높다. (복잡한 문장은 전환율 약 절반)

### 1-2. 모바일 & 속도
- 랜딩 트래픽의 **약 80% 이상이 모바일**이다.
- **로딩 2초 이내**일 때 전환율이 15% 정도 더 높다.
- 5초 → 1초로 줄이면 전환율이 **약 3배** 좋아진다는 연구가 있다.
- 나쁜 경험 한 사용자의 **88%는 다시 안 온다**고 인식된다.

### 1-3. 신뢰 & 사회적 증거
- **후기·사례·로고**가 있는 페이지가 전환에 유리하다. (상위 페이지의 37%가 포함)
- **구체적인 수치**가 있는 후기("시간 40% 절감")가 "좋아요"보다 훨씬 효과적.
- **93%**가 구매/문의 전에 리뷰를 본다.
- 신뢰 요소를 CTA 근처에 두면 전환 **15~34%** 향상 사례가 있다.

### 1-4. 문의/연락
- 폼 필드를 **4개 → 3개**로 줄이기만 해도 전환 **50%** 증가 사례가 있다.
- **이름, 이메일, 메시지**만 있어도 충분하다는 사례가 많다.
- **라이브챗/챗봇**이 있으면 문의 전환 **20%+** 증가 사례가 있다.
- 개인정보 처리 방침·이용 목적을 **쉽게 읽을 수 있게** 쓰면 제출률이 올라간다.

### 1-5. B2B·셀러 도구 사이트 특화
- **50ms** 만에 사이트 신뢰도를 판단한다는 연구가 있다.
- **75%**는 회사 신뢰도를 “웹사이트 디자인”으로 본다.
- 구매 결정에 **4~6명**이 관여하므로, 기술 담당·재무·경영진 모두가 이해할 수 있는 구조가 좋다.
- **가격 공개**, **데모/체험 바로가기**, **로드맵·정책** 등 **투명성**을 보여주는 게 중요하다.

## 2. 우리 웹사이트 진화를 위한 할 일 (우선순위)

### Phase 1: 당장 손댈 수 있는 것 (콘텐츠·구조)

| 할 일 | 고객 니즈 | 구체적 액션 |
|-------|-----------|-------------|
| **메시지 단순화** | 10초 안에 “나한테 필요한지” 판단 | 홈 첫 화면 문장을 **한 줄 가치 제안**으로 정리. 기능 나열 줄이고 “혜택” 위주로 재작성 |
| **CTA 하나로 모으기** | 명확한 다음 행동 | 페이지당 **메인 CTA 1개** (예: “무료 체험” 또는 “IR 미팅”). 버튼 문구를 행동형으로 (“지금 시작하기”, “데모 보기”) |
| **사회적 증거 추가** | 신뢰·검증 | 고객사 로고, **구체적 수치가 있는 후기 2~3개**, “언론 보도” 섹션 또는 뱃지. 가능하면 Contact/데모 요청 근처에 배치 |
| **문의 폼 단순화** | 문의 전환율 | Contact 폼을 **이름·이메일·메시지** 3개 필드로 축소. 제출 버튼 문구를 “문의 보내기” 등으로 명확히 |
| **푸터·정책 링크** | 투명성·신뢰 | 개인정보처리방침, 이용약관 페이지 추가 후 푸터에 링크. 짧은 문장으로 “어디에 쓰는지” 설명 |

### Phase 2: 경험·성능 (1~2주 단위)

| 할 일 | 고객 니즈 | 구체적 액션 |
|-------|-----------|-------------|
| **모바일 점검** | 80%+ 모바일 트래픽 | 실제 폰에서 모든 페이지·폼·버튼 터치 영역 확인. 스크롤 후에도 CTA 보이게 (sticky 등) |
| **로딩 속도** | 2초 이내 기대 | 이미지 WebP/최적화, 불필요 스크립트 제거. `npm run build` 후 번들 크기 확인 |
| **접근성** | 모든 사용자 | 이미지 `alt`, 제목 단계(h1→h2), 버튼/링크 명확한 이름. 색상 대비 확인 |
| **SEO 기본** | 검색 유입 | `index.html` title·description을 “ZeroSeller – 한 줄로 설명” 형태로. 주요 페이지마다 메타 설명 |

### Phase 3: 고도화 (선택)

| 할 일 | 고객 니즈 | 구체적 액션 |
|-------|-----------|-------------|
| **콘텐츠 분리** | 수정 용이·다국어 | 문구를 JSON/마크다운으로 빼고 컴포넌트에서 불러오기. 나중에 번역·A/B 테스트에 활용 |
| **문의 연동** | 실제 전환 | Contact 폼 제출 시 이메일 전송(Formspree, Netlify Forms 등) 또는 슬랙/노션 연동 |
| **챗봇/라이브챗** | 문의 전환 20%+ | 티켓팅·상담 링크 또는 간단 챗봇(예: 채팅 위젯) 추가 |
| **데모/가격 명확화** | B2B 신뢰 | 데모 페이지에 “무엇을 보여주는지” 1~2문장. 요금제는 가능한 범위에서 공개 |

## 5. 적용 완료 (컨셉·슬로건 업데이트)

다음 프롬프트가 반영되었습니다.

| 항목 | 변경 내용 |
|------|-----------|
| **서비스 컨셉** | Chat-to-Action → **Zero-Click Intelligence** / 능동형 에이전트. "대표님은 결정만 하세요. 실행과 분석은 AI가 보고합니다." |
| **가치 제안** | Zero Learning + **Zero Search** (찾아보지 않아도 됨). AI가 정보를 찾아와 차려주는 비서 모델 |
| **대시보드 UI** | 모니터링 → **AI 전략 리포트**. ① AI 분석 코멘트 ② 원클릭 승인 버튼, 결정(Decision) 중심 구성 |
| **성공 시나리오** | "상품 등록 완료" → **AI 분석 보고 → 대표님 클릭 승인 → 매출 최적화** 선순환 |
| **AI 마켓 가디언** | 경쟁사 활동(가격·리뷰 키워드·광고 소재) 실시간 크롤링·브리핑, 대화형 보고 예시 추가 |
| **기술 스택** | LangGraph → **Self-Reflecting Strategy Loop** (자율 학습형 피드백 루프), Flutter 대시보드, Kafka 실시간 데이터 |
| **문의 폼** | 이름·이메일·메시지 3필드로 단순화 (전환율 개선) |
